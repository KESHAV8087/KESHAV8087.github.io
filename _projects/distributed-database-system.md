### Overview
Developed a distributed database system focusing on **Two-Phase Commit (2PC)** protocols and **InnoDB**-style recovery mechanisms to ensure data integrity.

### Key Features
* **Concurrency Control:** Implemented multi-version concurrency control (MVCC) to handle high-traffic read/write operations.
* **Fault Tolerance:** Designed a WAL (Write-Ahead Logging) system to ensure 100% recovery after unexpected system crashes.
* **Performance:** Optimized query execution plans, reducing latency by 20% in benchmark testing.

### Tech Stack
* **Language:** Java
* **Concepts:** Distributed Systems, Transaction Management, Big O Optimization
